applied accessibility

the alt attribute

use the alt attribute so image info can be read by screen readers

use an empty alt attribute for images with explanation text, or images without meaning such as decoration and background images

headings

screen readers can be set to read only the headings on a page so the user gets a summary

choose header tags based on semantic meaning, not just font size

lower ranked headings imply subsections. don't jump from h2 to h4 - edit sizing with css

only use one h1 tag per page

semantic tags

these include main, header, footer, nav, article and section

assistive technologies can access this information to provide better page summary or navigation options to their user

only use one main element per page - don't use it for items that repeat across pages such as navigation links

main also offers assistive devices "jump to main content" functionality

use article to wrap independent, self-contained content such as blog entries, forum posts and news articles

div - groups content | section - groups related content | article - groups independent, self-contained content

header is used to wrap introductory information or navigation links for its parent tag and works well around content that's repeated at the top on multiple pages

header also offers assistive devices "jump to header" functionality

nav wraps around main navigation links, but not footer links

footer is primarily used to contain copyright information or links to related documents that usually sit at the bottom of a page and also has the "jump to" feature

audio content

use audio tags around audio content

audio content also needs alternative text - usually nearby text or link to a transcript

supports the controls attribute

figure and figcaption

used to wrap visual representations, such as charts, and their captions

adds both semantic info and a text explanation

forms

use the label tag to wrap the text for a specific form control item, usually the name or label for a choice

the for attribute on a label tag explicitly associates that label with the form control and is used by screen readers - must be same value as form control id

use fieldset tag on radio buttons to show that they are grouped together - use legend tag for description

don't use these if selection is obvious, such as gender - label with for will suffice

Choose one of these three items:

accessible date picker

input type, introduced in HTML5

for older browsers, the type will default to text, so show users the expected date format in the label or placeholder text

time and datetime

wrap dates or times with the time element and datetime attribute

datetime value is accessed by assistve devices and avoids confusion for informally stated dates

CSS

use CSS to visually hide content meant only for screen readers, such as when a screen reader needs a chart in table form, by pushing it out of the viewport

.sr-only { position: absolute; left: -10000px; width: 1px; height: 1px; top: auto; overflow: hidden; }

note that display: none and visibility: hidden hide the content for everyone, and 0px values remove item from flow of page making it unreadable by screen readers

color

use high contrast text - at least a 4.5 to 1 contrast ratio for normal text

check using online tools such as the mui color tool

as we know, don't just use color to convey important info - screen readers can't read this - use text alternatives

consider color blindness by shading darker colors (with blues, violets, magentas, reds) and tinting lighter colors (with oranges, yellows, greens and blue-greens) - try hsl() for example

there are different kinds of colorblindness to consider, the most common being distinguishing shades of green. some online color picking tools include visual simulations of how colors appear for different types of colorblindness

links

screen readers read the link text so make it meaningful - avoid 'read more' and 'click here'

use accesskeys to make navigation more efficient for keyboard only users - also works on buttons and form controls

tabindex attribute

links and form controls automatically recieve keyboard focus

tabindex="0" adds keyboard focus when a user tabs through page to elements such as div, span and p

tabindex="-1" indicates that it is focusable but not reachable by keyboard

tabindex="1" or higher ("2", "3"...) sets the tab order of the elements - note this can override the default order so use carefully
