applied accessibility

the alt attribute

use the alt attribute so image info can be read by screen readers

use an empty alt attribute for images with explanation text, or images without meaning such as decoration and background images

headings

screen readers can be set to read only the headings on a page so the user gets a summary

choose header tags based on semantic meaning, not just font size

lower ranked headings imply subsections. don't jump from h2 to h4 - edit sizing with css

only use one h1 tag per page

semantic tags

these include main, header, footer, nav, article and section

assistive technologies can access this information to provide better page summary or navigation options to their user

only use one main element per page - don't use it for items that repeat across pages such as navigation links

main also offers assistive devices "jump to main content" functionality

use article to wrap independent, self-contained content such as blog entries, forum posts and news articles

div - groups content | section - groups related content | article - groups independent, self-contained content

header is used to wrap introductory information or navigation links for its parent tag and works well around content that's repeated at the top on multiple pages

header also offers assistive devices "jump to header" functionality

nav wraps around main navigation links, but not footer links
